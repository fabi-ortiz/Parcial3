Notas de clase

  • Input type="text"- campos de texto de una sola linea
  • Input type submit- recoge la información y procesa el archivo php
  • Cols- es para los caracteres que podemos escribir
  • Rows – es para definir los renglones en los que se pueden escribir
  • Input type date- para poner la fecha y aparece un micro calendario
  • Input type radio – sirve para seleccionar una de varias opciones
  • Input type checkbox- sirve para seleccionar varias opciones
  • Input type button- se usa para eventos o funciones con java script
  • Input type reset- sirve para limpiar los campos del formularios
  • Input type color- sirve para crear un botón donde se pueda seleccionar el color
  • Input type range- sirve para seleccionar un rango, se le puede colocar un mínimo y un máximo con números enteros
  • Input type search- sirve para crear el botón de buscar
  • Input type number- sirve para crear un botón donde puedas seleccionar o poner un numero
  • Input type tel- sirve para agregar un numero de teléfono, pero se tiene que agregar un pattern([0-9]{3} [0-9]{7} ) y para que sa obligatorio se pone required

RESUMEN

Estos botones se realizan sin la necesidad de crear un archivo de css, tienen la misma funcionalidad que los que se pueden crear con css, solo que se ven menos esteticos. Estos se crean con los campos input y de estos tiene diferentes tipos, como los que estan en la seccion de notas de clase

Formularios en CSS

IDEAS PRINCIPALES

La base se crea con el html, haciendo una tabla. Utilizando los elementos de la tablas que ya conocemos.

Con el archivo css, lo acomodaremos y lo haremos mas esteticos para poder hacer que se vea como la imagen.

Notas de clase

Al unir los diferentes input podemos editarlos de la misma manera, para poder ponerles un hover y transicion
input[type="text"],input[type="email"]{
border-radius: 0;
border: 2px solid transparent;
padding: 3px;
}

input[type="text"],input[type="email"]:focus{
outline: 0;
border-radius: 0;
/*border: 2px solid plum;*/
box-shadow: 1px 1px 2px rgba(137, 43, 226, 0.65);
}

input[type="submit"]{
border-radius: 0;
border: 0;
background-color: palevioletred;
color: black;
}

input[type="submit"]:hover{
background-color: rgba(114, 49, 71, 0.751);
}
Con esto podemos hacer que el cuando el cursor seleccione algo este cambie de color

Tablas

IDEAS PRINCIPALES

Las tablas en html, consta de celdas de tabla dentro de filas y columnas

Antes de la creacion de css los elementos html table eran frecuentemente utilizados para el posicionamiento de una pagina.

Rowspan- idica el numero de filas que ocupara la celda. Por defecto ocupa una sola fila /td rowspan=3>, la celdaocupara el alto de 3 celdas normales

Colspan- indica el numero de columnas que ocupara la celda. Por defecto ocupa una solo columna. /td colspan=3>, quiere decir que la celda actual se extiende en el ncho de dos celdas

Notas de clase

    Elementos de una tabla
  • /table>- define a una tabla
  • /thead>- encabezado de la tabla
  • /th>- celda del encabezado de la tabla
  • /tbody>- cuerpo de la tabla
  • /tr>-renglon de la tabla
  • /td>- celda de datos
  • /caption>-usando para insertar captions
  • /tfoot>- footer de la tabla

RESUMEN

Con este elemenoto podemos crear tablas a partir de un HTML y acomodarlo esteticamente con un CSS. podemos convinar dos celdas o mas si el resultado es el mismo o la informacion es la misma, como se muestra en la imagen.

Especificidad

IDEAS PRINCIPALES

Es el medio por lo cual los navegadores deciden que valores de propiedad CSS son los mas relevantes para un elelemneto y por lo tanto se aplicaran.

Solo se aplica cuando el mismo elemento es el objetivo de varias declaraciones, según las reglas de CSS, los elementos dirigidos directamente siempre tendran prioridad sobre las reglas que un elemento hereda de su antepasado

Cuando varias declaraciones tienen la misma especificiad, la ultima declaracion encontrada en css se aplica al elemento.

Igualdad especificidad : si la misma regla se escribe dos veces en la hoja de estilo externa, estonces la reglainferior a la hoj DE ESTILO esta mas cerca del elemento que va a disenar u, por lo tanto, se aplicara:siempre la ultima regla.

Selector universal: el selector universal y los valores tienen una especificidad de 0. *, el cuerpo * y similares tienen una especificidad cero. Los valores heredados tambien tienen una especificidad de 0

La excepcion !importan
Cuando se usa esta regla !important en una declaracion de estilo, esta declaracion anula cualquier otra declaracion
Sin embargo !important es una mala practica y debe evitarse porque dificulta la depuracional romper la cascada natural en sus hojas de estilo

Notas de clase

    JERARQUIA:
    Hay cuatro categorias que definenn el nivel de especificidad de un selector
  1. Estilos en linea:un estilo en linea se djunta directamenete al elemento que se va a diseñar
  2. ID es un identificador unico para los elelemtnos de la pagina,como #navbar
  3. Clases, atributos y pseudoclases: en esta categoria incluye clases,[atributos] y pseudoclases como hover, focus, etc
  4. Elementos y pseudo elementos esta categoria incluye nombres de elementos y pseudoelementos, como h1,div,:before y :after

Colores en CSS

IDEAS PRINCIPALES

sRGB- Standard Red, green, blue sintasis extractive, el color no existe sin la luz

Tienen 256 valores del rojo, verde y azul, que son 16,777,216 colores
Existen mas de 4 formas de representar colores
• Keywords
• Hexadecimal
• RGB
• HSL
• HWB

Color y pagina web :
Seleccionar una paleta de colores correcta es importante para poder emitir bien el mensaje, ya que los colores tiene una psicologia

Pautas a tener en cuenta

• Identifica el publico objetivo del proyecto
• Transimir sensaciones, ya que los colores estimulan nuestro cerebro, pero debemos tener
cuidado por que mucha de estas asociaciones tienen un fundamento cultural
• La armonía de colores y el equilbrio
• Colores de la misma gama cromática
• Colores complementarios
• Colores opuestos (adobe color)
• No abusar de los colores
• Combina colores claros y colores obscuros para lograr un buen contraste

Notas de clase

  • Keywords:
    palabras clave, son los nombres de los colores
  • Hexadecimales:
    Trabaja de 16 caracteres (123456789ABCDEF) y se convinan para formar algun color. El cero representa que no hay color, se pone en negro y el F se pone en su maxima intensidad. Se debe poner siempre # para que sean 7 caracteres ejm- #00FF00, dos rojos, dos verdes y dos azules
  • RGB y RGBA:
    Red (0-255), green(0-255), blue(0-255), alfa (0-1), todos en su maxima intensidad da el color blanco y todos en su minima intensidad se pone en negro
  • HSL Y HSLA:
    Hue, Saturation, Lightness
    Hue (0* - 356*) Saturation (0% - 100%) Lightness(0% - 100%)
  • HWB:
    Hue, White, Black

RESUMEN

Los colores los utilizamos para poder mejorar esteticamente una pagina web, pero es muy importante saber escoger una buena paleta de colores, ya que al tener unos solores muy intensos puede cansar al usuario, asi que debemos dejar colores claros y calidos. Para poder sacar una paleta de colores se puede sacar desde una aplicacion o del mismo circulo cromatico, ya sea con colores complementarios o con la misma gama de colores

Pseudo-elementos, After y Before

IDEAS PRINCIPALES

Pseudoelementos ::after :: before:

Permiten insertar contenido en una pagina desde CSS sin que tenga que estar en el html El resultado no esta en el documento , pero aparece en la pagina como si lo estuviera

AFTER El selector:
after inserta algo despues del contenido de cada uno de los elementos seleccionados

BEFORE:
Inserta algo antes del contenido seleccionado

Notas de clase

    Se debe usar la propiedad conten
  • Una cadena de caracteres:

    conten:”texto”. Deben estar especialmente codificados como una entidad Unicode
  • Una imagen :

    conten: url(image.jpg):- la imagen se inserta en sus dimensiones exactas y no se puede cambiar el tamaño
  • Nada: content:

    ””;- útil para insertar formar y lograr crear figuras .

RESUMEN

Con estos pseudo-elementos, podemos hacer que el cursor selecione ya sea antes o despues la informacion, tambien con ellas podemos crear diferentes figuras como las que se muestran en la image, ya sea estaticas o con animacion

Header

IDEAS PRINCIPALES

Debe ser ordenada y limpia para que pueda ser visualizada con un solo vistazo y no de lugar a confusion.

Es un espacio generico que tendra el mismo diseno en todas las paginas web, primero hacer una lista para ver los elementos que tendra

Elementos que pueden ir: logotipo, menu, buscador, datos de ccontacto, redes sosciales, carritos de compra, etc.

Notas de clase

  • Logotipo, solo en la Esquina superior izquierda o en medio
  • Menu o nav: siempre en función del proyecto y de su estructura, se pude colocar un solo menú o dos.
  • Buscador: buscadores internos en la pagina
  • Datos de contacto: lo que podemos incluir es el teléfono y el email .
  • Redes sociales: se pueden incluir en el header se puede poner solo los iconos pero solo 2
  • Tienda en línea: se refleja en la web por el carrito de compra, el pedido, lista de deseos y los botones de comprar
  • Botones desatacados: para enfatizar un servicio
  • Barras fijas: puede mostrar anuncios, descuentos, cupones
  • Selector de idioma: si el sitio web es milti lenguaje un elemento obligatorio en el header es un selector de idioma
  • Formulario de suscripción : no es recomendable ya que ocupa demasiado espacio

RESUMEN

El header siempre se debe de ver ordenado y limpio, para no confundir al usuario. Tampoco debe estar saturado de informacion ni con colores muy intensos debe llevar un menu para que el usuario se pueda desplazar libremente en el sitio web, debe de tener una altura adecuada para que no tape parte del cuerpo del sitio.

Footer

IDEAS PRINCIPALES

Es un espacio generico que suele tener el mismo diseño en todas las paginas de un sitio web.

Planificar cada uno de los elementos que se van a inculir en el footer es parte del proceso de diseño de un Proyecto web

Notas de clase

  • Textos legales:
    Son elementos obligatorios en el footer de toda web y no hay lugar para interpretaciones
  • Cookies:
    Es también uno de los textos legales obligatorios
  • Redes sociales:
    Permiten una vía de contacto directa entre la marca y las visitas en la web, se pueden añadir los botones de cada plataforma o utilizar. Widgets para mostrar las ultimas publicaciones
  • Copyright:
    Los créditos del diseñad y los autores del material grafico como fotos, ilustraciones o videos, instituciones que colaboran con el proyecto, si es el caso.
  • Datos de contacto:
    Son una información muy útil para que las vistas se pongan en contacto con la empresa o negocio.
  • ellos de confianza:
    Es un distintivo que otorga la asociación de internet MX a sitios web que pertenecen a empresas, organizaciones, instituciones y personas identificadas y comprometidas con la generación de confianza en línea.
  • CTA(call to action):
    Es una imagen o texto que invita a los visitantes a realizar una acción.
  • Sitemap o mapa del sitio:
    Son archivos en los que se proporciona información sobre las paginas, los videos y otros archivos del sitio, los sitemaps informan que archivos de un sitio son importantes para el autor e incluyen datos importantes sobre ellos
  • Formularios de suscripción o newsletter:
    Es un formulario ubicado en cualquier pagina de un sitio web o blog donde los usuarios pueden completar los campos con sus datos para recibir correos electrónicos sobre temas relacionados con su interés

RESUMEN

En el footer, podemos agregar toda la informacion que no podemos poner en el header. Hay que poner mucha antencion respecto a los textos legales, ya que sus multas son muy caras y tambien colo los sellos de sefuridad, hay que verificar que sean verdaderos.
